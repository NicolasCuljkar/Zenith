Je développe une application de gestion financière personnelle.

Je veux implémenter une fonctionnalité permettant de connecter l'application aux comptes bancaires des utilisateurs via une API (bridgeapi.io)

Objectifs :

Permettre à l’utilisateur de lier plusieurs comptes bancaires, provenant d’une ou plusieurs banques

Mettre en place un l'api sécurisée pour récupérer les données financières

Récupérer et synchroniser en temps réel (ou quasi temps réel) :

le solde des comptes

les transactions (revenus et dépenses)

Stocker et structurer ces données côté backend

Créer une page dédiée affichant :

le solde total consolidé

le détail des soldes par compte

les flux (revenus / dépenses)

Contraintes techniques :

Authentification sécurisée (OAuth2 ou équivalent)

Gestion des tokens et refresh tokens

Gestion des erreurs API (timeouts, accès refusé, etc.)

Support de plusieurs fournisseurs bancaires

Architecture scalable et maintenable
